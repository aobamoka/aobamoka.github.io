网络是怎样连接的读书笔记1

网络是怎样连接的 读书笔记1

前言

想卷进外企只能提前多准备点东西了，正好计网学的也不是很行，先整两本入门书看看吧。后续可能还会更《图解TCP/HTTP》。

1.1 生成HTTP请求消息

1.1.1 输入网址

网址全称为URL(Uniform Resource Locator), 网络资源定位符.

URL开头的文字, 如”http:”, “ftp:”, “file:”, “mailto:”等, 表示浏览器应当使用的访问方法, 或者说访问时使用的协议类型. 如访问Web服务器时应当采用HTTP协议, 访问FTP服务器时应当采用FTP协议.

URL根据使用的协议不同包含了不同的信息, 如访问Web服务器或FTP服务器时, 应当包含服务器域名和访问的文件路径名, 而发邮件时应该包含收件人的邮件地址. 此外还可能包含用户名密码, 服务器端口号等.

示例如下(其中用户名, 密码, 端口号, 文件名通常省略, 形成我们平时看到的URL样式):

http://user:password@www.glasscom.com:80/dir/file1.htm

1.1.2 解析URL

浏览器的第一部工作就是对URL进行解析, 来获得应该访问的目标位置. 浏览器对URL解析时, 先将上个示例的URL拆分成几个元素

  • http: : 访问使用到的协议
  • // : 后面的字符串代表服务器名称
  • www.glasscom.com : 服务器名称
  • 后续: 文件路径名

1.1.3 省略文件名

http://www.lab.glasscom.com/dir/

服务器上会事先设置好省略文件名时应该访问的文件,通常为index.html或default.htm

http://www.lab.glasscom.com/dir

根据dir在目录下到底是文件还是目录来分别当作文件或目录处理.(无法创建两个名称相同的文件或目录)

1.1.4 HTTP基本思路

HTTP协议传递过程：

  • 客户端向服务器发送请求消息，包含信息“对什么”和“进行怎样的操作”。“对什么”即为URI包含的信息，“什么操作”有以下几种：
    • GET：获取URI指定的信息
    • POST：从客户端向服务器发送表单中填写的数据（需要用户输入）
    • HEAD：和GET基本相同，不过只返回HTTP消息头，用于获取文件最后更新的时间等属性信息
    • OPTIONS：通知或查询通信选项
    • PUT：替换URI指定的服务器上的文件
    • DELETE：删除URI指定的服务器上的文件
    • TRACE：将服务器收到的请求和头部直接返回给客户端。用于在使用代理的环境中检查改写请求
    • CONNECT：使用代理传输加密消息
  • Web服务器对其中的内容进行解析，完成相应的工作，并将结果放在响应消息中。响应消息头有一个状态码，表示操作的执行结果是成功还是发生了错误。
  • 客户端收到后，浏览器读出所需的信息并显示在屏幕上。

由于网络安全等问题，实际上广泛普及的只有GET与POST。

1.1.5 生成HTTP请求消息

请求消息第一行为请求行，通过这一行可以大致了解请求的内容

GET请求消息示例

GET /cgi/sample.cgi?Field1=ABCDEFG&SendButton=SEND HTTP/1.1

若干行头字段

POST请求消息示例

POST /cgi/sample.cgi HTTP/1.1

若干行头字段

Field1=ABCDEFG&SendButton=SEND

头字段包含额外的详细信息，如日期、客户端支持的数据类型、语言等。

消息体包含需要发送的数据（GET请求由于不需要发送数据，因此消息体中不需要填写数据）

1.1.6 服务器的响应消息

响应消息格式：

HTTP/1.1 200 OK

Date: Wed, 21 Feb 2007 09: 19: 14 GMT

...

<空行>

<html>

<head>

<消息体>

第一行为状态码和响应短语，用来表示请求执行结果是成功还是出错，下面是状态码表示的大致结果：

  • 1xx：告知请求的处理结果和情况
  • 2xx：成功
  • 3xx：需要进一步操作
  • 4xx：客户端错误
  • 5xx：服务器错误

网页中包含图片时，会在网页相应位置嵌入表示图片文件的标签的控制信息，浏览器在显示文字时搜索相应的标签，在屏幕上留出相应的空间并再次访问Web服务器请求图片文件。

一条URL只能获取一个文件。如果需要获取多个文件，必须对每个文件单独发送一条URL。

1.2 向DNS服务器查询IP地址

1.2.1 IP地址

IP地址被子网掩码分为网络号与主机号两个部分。

主机号全0：表示整个子网。

主机号全1：表示向整个子网发送包，即“广播”。

1.2.2 IP地址与域名并用的理由

单用IP地址：难以让人记住

单用域名：字符串相比比特串更难储存，并且对路由器负担更大

1.2.3 查询IP地址

域名解析：使用本机上的DNS客户端，调用Socket包的解析器方法向DNS服务器请求解析域名获取IP地址。

1.2.4 通过解析器向DNS服务器查询

1.2.5 解析器内部原理

第一步：解析器生成查询消息。

第二步：委托给操作系统内部的协议栈发送消息给DNS服务器。

第三步：DNS进行查询并将返回结果返回给解析器。（DNS的IP地址由用户在电脑里手动设置或自动获取）

1.3 DNS查询

1.3.1 DNS查找方式

来自客户端的查询消息所包含的信息：域名、邮件服务器名称（@后面的部分）；Class（网络类型，现在因为只剩下互联网所以弃用，只有IN）；记录类型（域名对应何种类型的记录）（A表示域名对应IP地址，MX表示域名对应邮件服务器）

DNS通过在记录中查表并对比上述三个项目的方式来找到对应的返回值。

1.3.2 域名层次结构

以www.lab.glasscom.com为例，层次结构大小从com到www依次递减。

一个域的信息作为一个整体存放在DNS服务器中，而一台DNS服务器可能存放多个域的信息。

在com、jp等域的最上层还有一个根域，域名应为www.lab.glasscom.com.但是最后的点经常省略。

上级DNS服务器保存着所有下级DNS服务器的信息。

每台DNS服务器都保存了根域的DNS服务器信息。

1.3.3 DNS查询

  1. 客户端访问最近的一台DNS服务器，请求查询信息www.lab.glasscom.com。
  2. DNS服务器本地没有保存与该域名对应的信息，因此向根域服务器发送查询消息。
  3. 根域根据域名结构解析出该域名属于com域，因此返回其所管理的com域的DNS服务器的IP地址。
  4. DNS服务器重复3步骤，直到最终获得www.lab.glasscom.com域的IP地址。
  5. DNS服务器将查询到的IP地址返回给客户端。

1.3.4 缓存

DNS服务器有缓存，可以记住一定时间内访问过的域名的信息，如果没找到域名，也会将“域名不存在”作为信息记录。

由于域名信息可能会更新，因此每条缓存都有时间限制，超过时限则会消失。

1.4 协议栈

1.4.1 数据收发流程

浏览器委托协议栈与Web服务器通信同样使用Socket库。

收发数据的操作大致分为四个阶段：1）创建套接字；2）将管道连接到服务器端的套接字上；3）收发数据；4）断开管道并删除套接字

1.4.2 创建套接字

应用程序调用Socket库的socket函数，协议栈返回一个描述符，它用来识别不同的套接字。

1.4.3 连接管道

应用程序调用Socket库的connect函数与服务器进行连接，需要指定描述符、服务器IP地址与端口号三个参数。

描述符与端口号的区别是，描述符是告诉本机的协议栈使用哪一个套接字与服务端的套接字进行连接，而端口号是告诉网络连接的另一方自身的套接字。

服务器所使用的端口号一般是事先规定好的。

本机使用的端口号在创建套接字时随机分配，并在连接时发送给服务器。

1.4.4 传递消息

应用程序调用Socket库的write与read函数与服务器传递消息。

1.4.5 断开连接

两边都可以主动断开连接。
