1. 内存管理

共4G虚拟内存（32位最大寻址空间）。

1. 内存各段的意义

从下向上为：

  • .text程序文件段：程序的二进制可执行代码，只读。
  • .data已初始化数据段：已初始化的全局变量和静态常量。
  • .bss未初始化数据段：未初始化的静态变量和全局变量。
  • 堆段：动态分配的内存，低地址向上增长。
  • 文件映射段：动态库、共享内存等，低地址向上增长。
  • 栈段：函数的参数值、局部变量、函数调用上下文。大小固定，可通过参数配置。高地址向下增长。

2. 分页的意义/为什么分页

  • 释放内存以页为单位释放，不会产生外部内存碎片。但会产生内部内存碎片。
  • 加载程序时不必一次性加载完，可以只在需要用到对应部分的时候再加载到内存中。
  • 内存空间不足时可将最近没使用的内存页面暂时置入磁盘，一次只需要置入几个页，解决交换效率低的问题。

2. 信号

用户、系统、进程发送给目标进程的信息，通知某个状态改变或系统异常。

常用信号：

  • SIGHUP控制终端挂起。
  • SIGPIPE向读端关闭的通道或socket连接中写数据。
  • SIGURG：socket连接上收到紧急数据。

3. Linux设计

  • MutiTask多任务：多个任务并发或并行。
  • SMP对称多处理：每个CPU地位相同，使用权限相同，共享一个内存，每个CPU都可以访问完整的内存和硬件资源。
  • ELF可执行文件链接格式
  • Monolithic Kernel宏内核：内核是一个完整的可执行程序，拥有最高权限。

4. I/O

通常涉及到磁盘、网络，需要IO接口支持数据交换。

I/O一般分为两步：

  • 等待数据准备好（设备管理器/网卡拷贝数据到内核缓冲区）
  • 操作系统将数据从内核缓冲区拷贝到用户。

同步IO与异步IO的区别就是要不要等待第二步。

1. I/O模型

1. 阻塞式IO

系统调用，如果没满足条件（如数据没准备好）就阻塞自己，直到内核准备好数据并拷贝到用户态，唤醒进程并操作数据。

阻塞时可让出CPU。

耗费时间，适合CPU并发低、时效性要求低的场合。

2. 非阻塞式IO

类似于自旋锁，如果没满足条件就轮询（一直进行系统调用问），直到内核准备好数据（IO第一步完成），然后等待其拷贝到用户态。

3. IO多路复用

调用select/poll，阻塞在这两个系统调用上，而非真正的IO系统调用上。

IO第一步完成时，select返回可读条件，调用recvfrom，等待完成IO第二步（阻塞）。

可注册多个进程，select监听所有注册好的IO。

优点：可同时监听多个描述符，且可以等待多个描述符就绪。

4. 信号驱动IO

通过sigaction系统调用安装一个信号处理函数，立即返回。

IO第一步完成时，内核给该进程产生一个ISGIO信号，进程接到信号后调用信号处理函数，在启动调用recvfrom，等待完成IO第二步（阻塞）。

优点：信号到达前不被阻塞，可继续执行程序。

5. 异步IO

通过aio_read系统调用，传递描述符、缓冲区指针、缓冲区大小和文件偏移，告知内核操作完后如何通知，然后继续执行。

内核收到后立刻返回，然后执行第一步、第二部，然后通知进程IO完成。

与信号驱动的区别：

  • 信号驱动通知的是可进行IO事件。（仍需等待第二步）
  • 异步IO通知的是IO完成。（不需要等待第二步）

6. 高效多路复用：epoll

红黑树/线性结构、epoll_ctl、事件驱动与回调函数/轮询、就绪队列/整个线性结构、返回全是就绪事件/还需要对返回值遍历、ET+LT/LT。

5. 控制台/常用命令

命令实现的原理

  • terminal终端：字符输入输出设备的抽象，在linux下是一种特殊的文件，键盘输入写入这个文件，对应的终端驱动程序会执行回显等工作。
  • shell：本质是命令解释软件。
  • 运行shell时，shell的stdin、stdout、stderr会打开并指向一个特定的terminal文件，使得我们的键盘输入实际上写到terminal里，但shell通过stdin从terminal文件得到。

执行方式：

  • ubuntu中默认的shell是bash，打开终端时就会启动bash。接下来输入的字符通过terminal传递给bash运行。bash执行指令的方式有三种
    • 绝对路径执行，指定执行的命令脚本的路径+命令文件
    • sh执行，用脚本对应的sh或bash接脚本执行
    • shell环境执行，在当前的shell环境中执行，使用.接脚本或source接脚本
    • 前两种方式会使用一个新的bash子进程执行命令，父进程睡眠。子进程执行完后回到父进程。

可看

规律：

  • -i一般为需要输出信息、提示时
  • -f与-i相对，一般为无需显示确认信息
  • -r/-R一般为递归执行操作

通配符：

  • ？可替代单个字符
  • *可替代任意多字符
  • [charset]可替代其中任意单个字符

1. 标准输入和参数

标准输入

  • 编程语言中scanf、readline这种命令（程序运行中）
  • 管道符 | 和重定向符 > < 将数据作为程序的标准输入
  • 本质上是个文件，被自动链接到终端的前台程序。程序切换时切换链接

参数

  • main函数传入的args字符数组（程序启动时）
  • $(cmd)读取cmd输出的数据作为参数

区分

  • 伴随程序启动时就附在命令后的属于参数，程序启动后才从终端输入的属于标准内容。
  • 命令能让终端阻塞，说明接收标准输入，否则不接受

2. 后台运行

cmd &：后台运行，但是关闭shell还是会关闭程序。因为shell下运行的程序都是shell进程的子进程。

(cmd &)：后台运行，关闭shell不关闭程序。相当于挂到systemd守护进程下，而不是shell进程。

3. sudo

sudo会让系统使用/etc/sudoers文件中规定的用户权限和环境变量。

不用sudo，使用的是/home/(username)/bin中的

sudo使用本地脚本文件时，用绝对路径，防止找不到。

4. 文件名组合

{s1, s2}, {s3, s4}为取两者交集，即{s1, s2, s3, s4}

{s1, s2}{s3, s4}为两者乘积，即{s1s3, s1s4, s2s3, s2s4}

5. 路径相关

  • cd -：返回刚才的路径
  • !$：替换成上一次命令最后的路径
  • !*：替换成上一次命令所有文件路径
  • 环境变量CDPATH中加入工作目录，cd在当前目录找不到时会自动到CDPATH中找

6. echo与!

echo：打印后面跟着的字符

参数：

  • -e：启动反斜杠转义字符
  • -n：不换行

是否加引号：

  • 单引号：不翻译任何东西，原样输出
  • 双引号：无视文件通配符，但是翻译$、\
  • 无引号：翻译所有可以翻译的

！：

  • !!：上一条命令
  • !$：上一条命令最后一个参数
  • !^：上一条命令第一个参数
  • !:-：上一条命令去掉最后一个参数
  • !*：上一条命令所有参数
  • !cp:n：上一条命令与指定参数
  • !n：history中编号为n的命令
  • !com：上一条以com开头的命令

7. ★ 查找

find/grep区别：

  • find搜索文件，完全匹配（通配符）
  • grep搜索文本，包含匹配（正则表达式）

命令：

  • find [path] -选项 参数：根据选项的参数要求在path中寻找文件。支持-and、-or、！表示与或非。
    • find [path] -name filename/“re”：根据path（绝对/相对路径）查找文件名为filename/文件名匹配re正则表达式的文件。
    • -type：按类型查找。f文件、d目录、l链接
    • -user：按用户查找
    • -group：按组查找
    • -time：按修改时间查找
  • grep (-额外选项) pattern path：在path（没有子目录）中找包含pattern的文件与行，且输出符合额外选项的规范
    • 额外选项：
      • -c(count)：只输出匹配行计数
      • -i(ignore)：不区分大小写
      • -h：查询多文件时不显示文件名
      • -I：查询多文件时只输出文件名
      • -n：显示匹配行与行号
      • -s：不显示不存在或无匹配的错误信息
      • -v：反转查找
      • -r：要求搜索子目录
      • -E：用正则表达式匹配
    • 特别的用法：带输出的命令 | grep (-选项) “pattern”。在输出中查找匹配pattern的行。相当于将命令的标准输出，通过匿名管道传入grep，作为grep的标准输入。

8. ls

ls [path default = ./] [-选项] [选项的额外参数]

  • -a(all)：显示所有文件与目录（包含隐藏）
  • -d(directory)：只列出目录
  • -l(long)：长格式显示文件和目录信息
  • -r(reverse)：倒序显示
  • -t(time)：按修改时间排序
  • -A：-a但不列出.与..
  • -F：列出文件后：可执行文件加”*”，目录加”/“
  • -R：递归显示子目录

9. ★ chmod

修改文件权限

说明：

  • u为owner文件拥有者，g为group与文件拥有者同一group，o为other其他人，a为三者（或ugo）。
  • +为增加权限，-为取消，=表示唯一设定。
  • r为可读取，w为可写入，x为可执行，X为仅当该文件是子目录或该文件已经被设置为可执行。
  • 也可用数字表示，三个三位数字，每个数字最高位表示r、次高位表示w、最低位表示x。
  • -R：递归
  • -c：有更改才显示更改动作
  • -f：无法更改不显示错误

示例：

  • chmod ugo+r 1.txt：所有人皆可读取
  • chmod ug+w,o-w 1.txt：文件拥有者与同一组人可写，其他人不可写
  • chmod -R a+r *：当前目录与子目录所有文件给所有人读取权限
  • chmod 777 1.txt：所有人对该文件有所有权限
  • chmod 4755 name：此程序有root权限

11. ★ sed

按行匹配文件并进行修改，将修改结果放在标准输出。不修改源文件。

sed [-hnv] [-e<script>] [-f<script文件>] [文件]

  • -e/-f：以选项中指定的script处理输入文本文件（也可以省略，带双引号就行了？）
  • -h：help
  • -n：只显示处理后的结果
  • -i：修改文件，在后面增加参数如-ie，会在文件名后面添加e，保存原来的内容，作为备份文件

动作：n动作string：指定第n行，指定动作，如果是写入动作则写入string，没有则不带string

  • a：新增

  • c：取代

  • d：删除

  • i：插入，在上一行出现

  • p：打印，与-n一起

  • s：取代，可搭配正则表达式

  • ‘/oo/p或d或{一组动作}’：所有包含oo的行

  • ‘s/old/new/g’：查找与替换（如果old与new中有/，可以用#等分隔符代替）

    • 不加g表示局部替换，只替换每一行第一个
    • 加了g表示全局替换，将所有匹配的都替换掉

  • 动作前加a,b表示指定的行范围，$为最后一行。

  • 可用分号连接多个动作。

示例：

  • sed -e 4a\newline filename：在filename第四行后添加一行，将结果输出到标准输出
  • nl filename | sed ‘3,$d’：删除第三行到最后一行
  • cat 2 | sed -e ‘4,$s#111#222#g’：将文件2中第四行到最后一行的所有111换成222

12. awk

awk是处理列的。sed是处理行的。

awk将每行按空格分割成数组，通过$n获取指定列，从1开始。

示例：

  • awk '{print $1}' filename：输出每行第一个元素
  • awk “NR==n” filename：输出第n行

13. 正则表达式

1. 字符匹配

  • [:alnum:]：字母和数字
  • [:lower:]：小写字母
  • [:upper:]：大写字母
  • [:digit:]：数字
  • [:xdigit:]：十六进制数字

2. 匹配次数

  • *：任意次
  • \?：0或1次
  • \+：至少一次
  • \{n\}：n次
  • \{m,n\}：m~n次
  • \{,n\}：至多n次
  • \{n,\}：至少n次

3. 位置锚定

  • ^：行首锚定
  • $：行尾锚定
  • ^PATTERN$：模式匹配整行
  • \< \b：词首锚定
  • \> \b：词尾锚定

4. 后向引用

括号分组，左起第n个左括号与其匹配的右括号中间匹配到的字符为第n组。后续可通过\n指定。

14. expr

  • 字符串长度：expr length "str"
  • 字符串子串：expr substr "string" [start] [len]。start为下标，从1开始。
  • 字符串中某字符第一次出现的位置：expr index "string" io。i或o第一次出现的位置，从1开始。
  • 整数运算：expr 30 \* 3：乘号必须转义，否则shell可能会误解。

15. ★ top

查看cpu占用。

选项：

  • -d n：每隔n秒更新一次，默认3秒
  • -b：批处理模式输出，一般与-n合用，重定向到文件中
  • -n：top命令执行的次数
  • -p pid：只查看指定pid进程
  • -u user：只监听指定用户的进程

top窗口交互：

  • P：按cpu使用率排序，默认
  • M：按内存使用率排序
  • N：按pid排序
  • T：按cpu累计运行时间排序
  • k：按pid给予某个进程信号
  • r：按pid设置某个进程优先级
  • q：退出

综合

kill：将指定信息送至程序，让其终止。

  • kill -l：列出所有可发送的信号
  • kill -9/-KILL pid：杀死pid线程（另：-1/-HUP为重新加载，-15/-TERM为正常停止）

ctrl+c/z

  • c：强制中断程序执行并释放内存。SIGINT。只对当前前台进程,和他的所在的进程组的每个进程都发送。
  • z：暂停程序，不释放内存。进程处于挂起态。SIGTSTP。
  • 另：d：不是发送信号，而是输入EOF
  • kill：SIGTERM。只有当前进程收到，可以被阻塞、处理、捕获。
  • kill -9：SIGKILL。无视任何处理直接杀死。

cp：复制文件

  • -i：覆盖前逐一询问
  • -f：覆盖无需确认
  • -r：递归复制文件夹
  • -p：还复制修改时间和访问权限
  • -l：不复制，只生成链接文件
  • -d：复制时保留链接
  • -v：将文件复制到指定目录下
  • -a：-dpr
  • scp：远程拷贝

rm：删除

  • -i：删除前逐一询问
  • -f：即使设置只读也删除而无需确认
  • -r：递归删除
  • 可组合，如-rf
  • rmdir：删除空文件夹

mv：移动/重命名

ps：显示当前进程状态

  • 查找指定进程：ps -ef | grep 进程关键字

终端：/dev/tty

黑洞文件：/dev/null（写多少都会消失）

wc：计算文件byte数、字数、行数。文件名缺省则读取标准输入

  • -c/–bytes/–chars：只显示byte数
  • -l/–lines：只显示行数
  • -w/–words：只显示字数

head/tail：头/尾显示文件，默认10行

  • 监视文件：只有tail有
    • -f：监视文件，持续读取
    • -F：即使文件位置移动、名字变了也能监视到
  • -c n：显示n字节
  • -n n：显示n行

  • –pid=PID：与-f合用，在PID进程死掉后结束

bg/fg

  • bg：后台任务调度到前台
  • fg：前台任务调度到后台

which/whereis：查找特定文件

  • which只查可执行文件

alias：给命令取别名，如alias la = ‘ls -a’

ifconfig：显示所有网卡信息

netstat：显示网络状况

6. 调试

1. 段错误

由于内存问题引起

  • 访问不存在的内存地址
  • 访问系统保护的内存地址
  • 访问只读的内存地址
  • 栈溢出
  • delete使用错误（删除了非new的内存、重复delete）
  • 数组越界

2. core

  • ulimit -c：查看core开关
  • ulimit -c [size]/unlimited 设置core大小
  • gdb [exec file] [core file]：运行后执行bt查看错误位置的堆栈信息
  • gdb -c [core file] -> file [exec file] -> bt：跟上面一样
  • x [virtual addr]：查看指定虚拟地址存储内容

3. gdb

  • gdb ./二进制可执行文件：进入gdb
  • run：直接运行程序
  • start：在main起始位置停下
  • list：查看调试行上下五行。
    • list xxx.c:20：查看该文件20行上下五行
  • step：逐步调试。进入函数内部
  • next：逐步调试。直接运行完函数
  • until xxx.c:20：直接运行到该行
  • continue：从当前位置开始运行
  • break xxx.c:20：打断点
  • info break：查看断点
  • delete [断点号]：删除所有断点/删除指定断点
  • backtrace：查看函数调用栈
